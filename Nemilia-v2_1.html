One-time install
  1. Download nemilia-extension.zip and unzip it anywhere
  2. Open chrome://extensions → turn on Developer mode
  3. Click Load unpacked → select the unzipped folder
  4. Open the extension Details → enable Allow access to file URLs
  5. Pin the extension from the 🧩 toolbar icon for quick access

About Nemilia AI
v2.1 · Chat Mode · Auto Workflow · Human-in-the-loop · Semantic vector RAG · MCP Tool Execution
Nemilia (Neh-mee-lee-ah) — from the Nahuatl verb meaning to think, to remember, or to imagine. Chosen because thinking and remembering are exactly what an LLM does when leveraging persistent memory and cross-run context.

A single HTML file that runs entirely in your browser. No server, no install, no account. Bring your own API keys.

Providers: Groq · OpenRouter · Gemini · Anthropic · OpenAI · Mistral · DeepSeek · Kimi (Moonshot) · xAI · Perplexity · Chrome AI (Gemini Nano) · Ollama · LM Studio · Jan · Llamafile
Search: Brave Search · Serper (Google) · Tavily AI · SearXNG (self-hosted)
Storage: File System API (Chrome/Edge) · IndexedDB (Firefox/Safari) · localStorage fallback
RAG: Semantic vector search (Transformers.js) + BM25 hybrid · PDF · DOCX · TXT · MD · CSV
Pipeline: DAG staggered execution · Human-in-the-loop (HITL) checkpoints · Drag-and-drop workflow builder · Role-based tool access · SLM prompt compression · SVG flow visualisation
Security: Hardened XSS esc() · CSP meta tag · AES-256-GCM · 200k PBKDF2 · stream error fallback · import validation · color sanitisation
New in v2.1: Chrome extension with offline capture queue · Captures queue with thumbnails, vision status, triage actions (→ Compose, → Chat, → Workflow, → Library) · Library replaces Documents — manage RAG sources, picker-only context selection · Promote to Library — move analyzed captures directly into RAG with one click · Compose attachment chips — attach captures as inline context cards · Inbox-owned image analysis — screenshots and page images analyzed in Captures, not shadow docs · Tags, search, batch routing · Per-agent web search (auto-detect / always / custom query) · Perplexity provider with live web search · Chrome AI (Gemini Nano) on-device provider · Auto-embed queue with persistent progress banner · Doc preview modal · Browser compatibility warnings · Sort controls on all card views · Provider switch identity injection in Chat · Mid-chat dismissible provider banner · Custom tag modal (apply / create / purge) · Vision model guardrail banner
v2.1 fixes: → Chat now attaches captures as visible chip in the Chat context bar with pre-filled instruction — no longer dumps raw text into the input · → Workflow button added to Captures triage row — attaches item to Compose and focuses the workflow dropdown · Extension image right-click simplified to single flat "Send Image to Nemilia" item with no submenu — always lands in Captures inbox · Capture-level processing badges now read from item state directly (Vision ✓ → BM25 → ⬡ Vector → ✓ RAG Ready) without requiring Library promotion · Auto-embed on ingest — text captures auto-chunk and queue for vector embedding on arrival; image captures auto-queue embedding after vision analysis · Vision token chip appears on image capture cards after analysis · Promote to Library carries over computed embeddings so re-embedding is skipped · activeChatCaptures state persists per chat session and resets correctly on new chat / switch / delete
